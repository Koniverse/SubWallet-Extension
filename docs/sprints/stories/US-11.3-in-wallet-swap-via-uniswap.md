## Goal

A user can swap EVM tokens via Uniswap — V3/V4 pools and UniswapX (Dutch-auction
intent) routing — from inside the wallet, with a live quote and single-confirm
execution.

## Background

Uniswap is the canonical EVM DEX; this provider covers its V3 and V4 pools plus
**UniswapX**, the intent-based Dutch-auction "Dutch Swap" route. It is integrated
as one **SwapService handler** (PRD [FR-8](../../PRD.md#functional-requirements), EPIC-2) — a new provider
is a new handler, never a new swap-UI branch — and quote/fee data is aggregated
via the Services SDK backend
([AD-24](../../ARCHITECTURE.md#architecture-decisions)). Uniswap is a **keyed
provider**: routing API access is proxied through the SubWallet backend per
[NFR-16](../../PRD.md#non-functional-requirements), never shipped in the bundle. Execution is an EVM
transaction.

Materializes [FR-105](../../PRD.md#functional-requirements). **Retroactive** — already shipped.

## Acceptance criteria

- [x] **AC-1** — **Given** an EVM pair tradable on Uniswap, **When** the user enters
  an amount, **Then** a quote (rate, fee, slippage, min received) is shown, sourced
  from the best of V3/V4/UniswapX routes.
- [x] **AC-2** — **Given** a quote, **When** the user confirms, **Then** the swap
  transaction is signed, submitted, and tracked in history.
- [x] **AC-3** — **Given** no route, slippage exceeded, or a stale quote, **When**
  the user swaps, **Then** a clear error is shown and nothing is submitted.
