## Goal

A user can swap between tokens that don't live on the same chain — the wallet
automatically composes a swap with a bridge (Swap→Bridge or Bridge→Swap) and
guides them through the multi-step route — so that they reach pairs no single DEX
lists.

## Background

This is the routing layer that sits **above** the per-provider handlers (e.g.
[US-11.2](US-11.2-in-wallet-swap-via-hydration-dex.md) and the other provider
stories US-11.3–11.11). When source and destination are on different chains, the
SwapService composes a route that chains a same-chain swap with a bridge step (XCM
or an external bridge, delegated to ParaSpell per
[AD-18](../../ARCHITECTURE.md#architecture-decisions)) and tracks each hop to
completion. It is one FR but a large capability — and it has distinct failure
semantics: a half-completed multi-hop must be **resumable**, never silently lost
between hops. That is why it is a separate, larger story (8 pts — a multi-system
integration across the swap and bridge engines) and not folded into a provider story.

Materializes [FR-110](../../PRD.md#functional-requirements). **Retroactive** — already shipped.

## Acceptance criteria

- [ ] **AC-1** — **Given** a cross-chain pair, **When** the user requests a swap,
  **Then** the wallet returns a multi-step route (swap + bridge) showing each
  step's direction, fee and estimated time before any signature.
- [ ] **AC-2** — **Given** a confirmed route, **When** the user executes it, **Then**
  each hop is submitted and tracked in order, and the UI shows per-step status
  (pending / done / failed).
- [ ] **AC-3** — **Given** a route that stalls after the first hop, **When** the
  user returns, **Then** the partial route is recoverable / resumable and clearly
  surfaced — funds are never reported as lost.
- [ ] **AC-4** — **Given** routing, **When** a step needs a bridge, **Then** it
  reuses the bridge engine (AD-18 / EPIC-13), not a bespoke per-pair path.

## Tasks

## Dev notes

### Architecture constraints

- [AD-18](../../ARCHITECTURE.md#architecture-decisions) — bridge construction is
  delegated to ParaSpell; routing consumes it, does not reimplement XCM.
- [AD-24](../../ARCHITECTURE.md#architecture-decisions) — route quotes flow through
  the Services SDK backend.

### Cross-story dependencies

- Builds on the per-provider handlers (US-11.1–11.7, 11.9–11.11) — composes them
  into multi-hop routes.
- Sibling: EPIC-13 (bridge-xcm) owns the bridge step this routing consumes.
