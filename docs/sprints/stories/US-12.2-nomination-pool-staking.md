## Goal

A user can join a nomination pool on Polkadot/Kusama — pooling their stake with
others under a single nominator — and bond-extra, claim rewards, unbond and
withdraw, so that holders below the solo-nomination minimum can still earn
staking rewards from the wallet.

## Background

Nomination pools let many small stakers share one nominator account, lowering the
effective minimum to participate in native staking. The mechanism is the engine's
`NominationPoolHandler` ([AD-22](../../ARCHITECTURE.md#architecture-decisions),
[US-2.3](US-2.3-earningservice-pool-handler-engine.md)) exposed over RxJS
subjects. This story is the pool-staking *screen*: it reuses the earning-list and
pool-handler subject fixture established by
[US-12.1](US-12.1-native-nomination-staking-relay-and-parachains.md) and adds the
join / bond-extra / claim / unbond / withdraw write flows specific to the pool
pallet.

Materializes [FR-115](../../PRD.md#functional-requirements). **Retroactive** — already shipped.

## Acceptance criteria

- [x] **AC-1** — **Given** a Polkadot/Kusama account, **When** the user joins a
  pool with an amount at or above the pool minimum, **Then** the join extrinsic
  submits through `NominationPoolHandler` and the pool position appears in the
  earning list.
- [x] **AC-2** — **Given** an active pool position, **When** the user bond-extra /
  claims rewards / unbonds / withdraws, **Then** each action submits the
  corresponding pool extrinsic and the position state updates from the subject.
- [x] **AC-3** — **Given** a join amount below the pool minimum or a pool that is
  not in an open/joinable state, **When** the user tries to join, **Then** the
  flow blocks with a clear validation error and submits nothing.
