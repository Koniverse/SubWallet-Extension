## Goal

A user can perform an intent-based cross-chain swap via Optimex from inside the
wallet — expressing a desired outcome and letting solvers settle it across chains
— with a live quote and single-confirm execution.

## Background

Optimex is an **intent-based cross-chain** swap provider: the user signs an
intent and off-chain solvers compete to fill it across chains, rather than the
wallet specifying an exact on-chain route. It is integrated as one **SwapService
handler** (PRD [FR-8](../../PRD.md#functional-requirements), EPIC-2) — a new provider is a new handler,
never a new swap-UI branch — and quote/fee data is aggregated via the Services
SDK backend ([AD-24](../../ARCHITECTURE.md#architecture-decisions)). Because it
settles across chains, it participates in the cross-chain routing layer
([US-11.8](US-11.8-cross-chain-swap-routing.md)); the bridge step that routing
composes around it is delegated to ParaSpell
([AD-18](../../ARCHITECTURE.md#architecture-decisions)).

The intent lifecycle (sign intent → solver fill → settlement, with the
asynchronous fill and partial-state semantics that implies) makes this a larger
integration than a same-chain DEX provider.

Materializes [FR-111](../../PRD.md#functional-requirements). **Retroactive** — already shipped.

## Acceptance criteria

- [ ] **AC-1** — **Given** an Optimex-supported cross-chain pair, **When** the user
  enters an amount, **Then** an intent quote (expected output, fee, slippage, min
  received, estimated time) is shown before any signature.
- [ ] **AC-2** — **Given** a quote, **When** the user confirms, **Then** the intent
  is signed, submitted to Optimex, and the fill is tracked in history through to
  settlement.
- [ ] **AC-3** — **Given** an intent that no solver fills (no route) or an Optimex
  outage / stale quote, **When** the user swaps, **Then** a clear error is shown
  and, on an unfilled intent, funds are not reported lost.

## Dev notes

### Architecture constraints

- [AD-24](../../ARCHITECTURE.md#architecture-decisions) — quotes/fees via the Services SDK backend.
- [AD-18](../../ARCHITECTURE.md#architecture-decisions) — the bridge step in cross-chain routes is delegated to ParaSpell; this provider consumes routing, it does not reimplement XCM.
- Plugs into SwapService (FR-8); no Optimex-specific swap-UI branch.

### Cross-story dependencies

- Builds on [US-2.4](US-2.4-swapservice-routing-engine.md) — the SwapService engine (FR-8) this handler registers into.
- Sibling: [US-11.8](US-11.8-cross-chain-swap-routing.md) — Optimex's cross-chain pairs feed the multi-hop routing layer, which owns the resumability semantics this intent flow relies on.

### Points justification

- 8 pts (§3a-bis) — a multi-system integration: an intent-based cross-chain
  provider whose asynchronous solver-fill lifecycle (sign-intent → fill →
  settlement, with unfilled-intent recovery) spans the swap engine, the
  cross-chain routing layer and the backend, beyond a single same-chain DEX
  handler.
