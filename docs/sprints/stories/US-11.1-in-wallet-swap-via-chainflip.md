## Goal

A user can swap natively across ecosystems — Bitcoin ↔ Polkadot ↔ EVM — from
inside the wallet via Chainflip, with a live quote and single-confirm execution.

## Background

Chainflip is a native cross-chain DEX that settles swaps directly between
Bitcoin, Polkadot and EVM assets. It is integrated as one **SwapService
handler** (PRD [FR-8](../../PRD.md#functional-requirements), EPIC-2); a new provider is a new handler,
never a new swap-UI branch. Quote/fee data is aggregated via the Services SDK
backend ([AD-24](../../ARCHITECTURE.md#architecture-decisions)). Chainflip is a
**keyed provider** — its API key is proxied through the SubWallet backend per
[NFR-16](../../PRD.md#non-functional-requirements), never shipped in the bundle. Because the provider settles
across chains, it participates in the cross-chain routing layer
([US-11.8](US-11.8-cross-chain-swap-routing.md)).

Materializes [FR-103](../../PRD.md#functional-requirements). **Retroactive** — already shipped.

## Acceptance criteria

- [x] **AC-1** — **Given** a Chainflip-supported cross-chain pair (e.g. BTC↔DOT),
  **When** the user enters an amount, **Then** a quote (rate, fee, slippage, min
  received, estimated time) is shown.
- [x] **AC-2** — **Given** a quote, **When** the user confirms, **Then** the swap
  is signed, submitted to Chainflip, and tracked in history to settlement.
- [x] **AC-3** — **Given** no route or a Chainflip outage / stale quote, **When**
  the user swaps, **Then** a clear error is shown and nothing is submitted.

- Keyed provider: API key proxied through the backend ([NFR-16](../../PRD.md#non-functional-requirements)), never bundled.
- Plugs into SwapService (FR-8); no Chainflip-specific swap-UI branch.
