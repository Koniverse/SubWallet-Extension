## Goal

Make every **feature that ends in a signature** work when the signer is a Ledger — transfer, XCM,
staking and earning, dApp requests, on-ramp. Each row is one feature discovering that a hardware
account is not a software account.

## Status

> **✅ done — all 14 rows below are settled**, delivered across releases 1.0.2 → 1.3.49 and later
> closes with no release of their own. It carries **no FR**: the features themselves belong to
> [EPIC-8](../epics/EPIC-8.md), [EPIC-12](../epics/EPIC-12.md), [EPIC-13](../epics/EPIC-13.md) and
> [EPIC-10](../epics/EPIC-10.md); what is recorded here is the **hardware branch** each of them
> needed.
>
> **`version_shipped: 1.3.49` is a representative anchor, not the whole set** — the most recent
> constituent with a provable release. The table is the full record.

## Scope

Folded in from the former one-issue-per-story maintenance ledger on 2026-07-23.

**Why these rows are here and not in the feature epics.** Each one is a defect in the *hardware*
path of a feature that already works for software keys: staking on Bifrost is fine until the signer
is a Ledger; XCM is fine until the signer is a Ledger. The shared cause is EPIC-16's, so the record
is EPIC-16's — the same reasoning that keeps
[US-16.1](US-16.1-ledger-hardware-wallet-signing.md)'s scope at *"how a device completes a
signature, not what is being signed"*.
