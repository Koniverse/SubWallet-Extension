## Goal

Decide **which pairs the wallet offers at all**, and measure whether anyone uses them. The catalogue
lives in the CMS and the backend, not in the extension bundle, so a pair can be added or withdrawn
without a release.

## Status

> **✅ done — all 4 rows below are closed `COMPLETED` with board `Status = Done`.**
>
> **`version_shipped` and `commit` are empty on purpose.** Every row ships **outside this
> repository** — the SubWallet CMS, the swap backend, and a BI dashboard — so no extension release
> carries them and no commit here makes them true: the third done-gate branch in
> [AGENTS.md](../../../AGENTS.md) rule 4. It carries **no FR**.

## Scope

Folded in from the former one-issue-per-story maintenance ledger on 2026-07-23. Its own story
because the catalogue is a **control surface, not a code path**: the same mechanism that lets
[EPIC-6](../epics/EPIC-6.md) toggle a chain at runtime lets swap withdraw a pair the moment a pool
goes thin, with no extension update in between.

**Out of scope:** the in-extension consequence of a catalogue change — which pairs the quote
pipeline then fans out to — is [US-11.13](US-11.13-swap-quote-pipeline-and-service.md). The open
tooling request, exporting and creating the pair files by hand
([#4209](https://github.com/Koniverse/SubWallet-Extension/issues/4209)), is in
[US-11.16](US-11.16-swap-flow-improvements.md) ([AGENTS.md](../../../AGENTS.md) rule 9).
