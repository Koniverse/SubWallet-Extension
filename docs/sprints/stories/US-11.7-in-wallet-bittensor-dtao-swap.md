## Goal

A user can swap TAO ↔ alpha and between alpha tokens across Bittensor subnets
from inside the wallet, via the subnet AMM, with a live quote and single-confirm
execution.

## Background

Bittensor's dTAO model prices each subnet's **alpha** token against **TAO**
through a per-subnet AMM. This provider covers TAO↔alpha and cross-subnet
alpha↔alpha swaps. It is integrated as one **SwapService handler** (PRD
[FR-8](../../PRD.md#functional-requirements), EPIC-2) — a new provider is a new handler, never a new
swap-UI branch — and quote/fee data is aggregated via the Services SDK backend
([AD-24](../../ARCHITECTURE.md#architecture-decisions)). Execution is a Substrate
extrinsic on the Bittensor chain; a cross-subnet alpha↔alpha swap composes two
subnet-AMM legs through the shared handler, not a bespoke UI path.

Materializes [FR-109](../../PRD.md#functional-requirements). **Retroactive** — already shipped.

## Acceptance criteria

- [ ] **AC-1** — **Given** a TAO↔alpha or cross-subnet alpha↔alpha pair, **When**
  the user enters an amount, **Then** a quote (rate, fee, slippage, min received)
  from the subnet AMM(s) is shown.
- [ ] **AC-2** — **Given** a quote, **When** the user confirms, **Then** the swap
  extrinsic is signed, submitted, and tracked in history.
- [ ] **AC-3** — **Given** insufficient subnet liquidity or a stale quote, **When**
  the user swaps, **Then** a clear error is shown and nothing is submitted.
