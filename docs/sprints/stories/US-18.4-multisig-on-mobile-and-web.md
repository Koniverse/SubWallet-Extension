## Goal

Bring the multisig capability the extension shipped in 1.3.74 to the **other two platforms** —
SubWallet Mobile ([#4697](https://github.com/Koniverse/SubWallet-Extension/issues/4697)) and the
SubWallet WebApp ([#4698](https://github.com/Koniverse/SubWallet-Extension/issues/4698)). Both are
open and unstarted.

## Scope

This is a **consolidated maintenance story**: it groups 2 tracker issues into one boundary,
replacing the former one-issue-per-story ledger. It materializes **no FR** — FR-149 and FR-150 are
fully owned by [US-18.1](US-18.1-multisig-account-creation-and-management.md) and
[US-18.2](US-18.2-pending-transaction-detection-and-approval.md), both `done` in 1.3.74, and a
platform port ships no new requirement. Full traceability is the table below and the
[consolidation note](../../notes/2026-07-22.md#c-epic-38-maintenance--multisig-merged-into-epic-18). **Not yet delivered** —
`assignee` / `commit` / `sprint` / `version_shipped` stay empty until it ships.

**Why it is its own story and not rows in US-18.1/US-18.2.** Those two are `done`; these two
issues ask for something they never delivered. Folding unfinished work into a shipped story is how
a `done` story comes to carry an unticked AC — the split the owner directed after the NFT
consolidation: completed work stays in the completed story, remaining work becomes its own story.

**The two targets sit on opposite sides of this repository.** #4698 is **in-repo** —
`packages/extension-web-ui` and `packages/webapp` — and the port has not started: the whole web UI
holds exactly one multisig reference (a type import in `packages/extension-web-ui/src/types/`)
against 40+ files in `packages/extension-koni-ui`; the multisig strings already in
`packages/webapp/public/locales/*/translation.json` arrived through the i18n sweep
([#4983](https://github.com/Koniverse/SubWallet-Extension/issues/4983)), not a feature port.
#4697 is **out-of-repo** — the mobile app lives in `Koniverse/SubWallet-Mobile`, so no commit in
this repository will ever close it, the same boundary
[US-1.6](US-1.6-platform-operations-and-out-of-repo-tooling.md) draws for backend and tooling work.
