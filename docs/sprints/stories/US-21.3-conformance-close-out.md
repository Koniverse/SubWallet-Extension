## Goal

Run the final five-layer conformance check after the history backfill (US-21.2) is
complete, graduate durable decisions from `docs/notes/` into CONTEXT.md and
LESSONS.md, regenerate STATUS.md from the current story state, and make
`npx koni-docs validate` exit zero as a repeatable CI gate.

## Background

After US-21.1 (contributor map) and US-21.2 (history backfill), the doc layers
should be internally consistent. But consistency is not yet *verified* — and
`koni-docs validate` may still fail on FR-reachability issues, sprint-file
coverage gaps, or STATUS.md regeneration errors. This story closes the loop:

1. **5-layer consistency check** — PRD FR → Epic story list → Story frontmatter →
   Sprint scope table → STATUS.md. Every level must agree.
2. **Hygiene sweep** — Run `koni-docs validate` and fix any remaining issues.
3. **Graduate notes** — Move durable decisions from `docs/notes/` into
   `CONTEXT.md` (append-only decision log) and `LESSONS.md` (recurring traps).
4. **Regenerate STATUS.md** — Run `koni-docs status` to produce the canonical
   kanban (RULE-5).
5. **Set the CI gate** — Ensure `koni-docs validate` exits zero and document
   the command in the pre-commit checklist.

## Acceptance criteria

- [ ] **AC-1** — **Given** the full set of doc layers, **When** `npx koni-docs validate` runs, **Then** it exits with code **0** **And** reports no errors.
- [ ] **AC-2** — **Given** the PRD.md, **When** every story's `prd_ref` is checked, **Then** every FR-N referenced resolves to an existing row in PRD's `Functional Requirements` section (L3 ID-graph integrity).
- [ ] **AC-3** — **Given** the epic files, **When** each epic's Stories table is compared against the actual story files in `docs/sprints/stories/`, **Then** every story listed exists **And** no story is orphaned (listed in no epic).
- [ ] **AC-4** — **Given** the sprint scope tables, **When** each story's `sprint` frontmatter is checked, **Then** every story assigned to a sprint appears in that sprint's scope table.
- [ ] **AC-5** — **Given** the STATUS.md file, **When** it is regenerated with `koni-docs status`, **Then** it reflects the current state of all 169 stories **And** contains no hand-edited artifacts.
- [ ] **AC-6** — **Given** the `docs/notes/` directory, **When** durable decisions and traps are identified, **Then** they are appended to `CONTEXT.md` (decisions) and `LESSONS.md` (traps/patterns) **And** the source note files are updated with a `(graduated)` marker.

## Dev notes — Architecture constraints

- RULE-5: STATUS.md is auto-generated — never hand-edit. Every regeneration replaces the file entirely.
- RULE-7: CONTEXT.md is append-only. Corrections use revision entries, not edits.
- The `docs/notes/` directory is a **scratch space** — durable content graduates OUT of it into the canonical doc files.
