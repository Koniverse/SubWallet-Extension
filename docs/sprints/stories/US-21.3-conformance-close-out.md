## Goal

Run the final five-layer conformance check after the history backfill (US-21.2) is
complete, graduate durable decisions from `docs/notes/` into CONTEXT.md and
LESSONS.md, regenerate STATUS.md from the current story state, and make
`npx koni-docs validate` exit zero as a repeatable CI gate.

## Background

After US-21.1 (contributor map) and US-21.2 (history backfill), the doc layers
should be internally consistent. But consistency is not yet *verified* — and
`koni-docs validate` may still fail on FR-reachability issues, sprint-file
coverage gaps, or STATUS.md regeneration errors. This story closes the loop:

1. **5-layer consistency check** — PRD FR → Epic story list → Story frontmatter →
   Sprint scope table → STATUS.md. Every level must agree.
2. **Hygiene sweep** — Run `koni-docs validate` and fix any remaining issues.
3. **Graduate notes** — Move durable decisions from `docs/notes/` into
   `CONTEXT.md` (append-only decision log) and `LESSONS.md` (recurring traps).
4. **Regenerate STATUS.md** — Run `koni-docs status` to produce the canonical
   kanban (RULE-5).
5. **Set the CI gate** — Ensure `koni-docs validate` exits zero and document
   the command in the pre-commit checklist.

Two stale artifacts were corrected, both in the sprint file rather than the layers it
checks: the scope table still showed US-21.2 as `🚧 in-progress` and claimed "175
stories" (now 174, after the duplicate deletions).

**AC-6 — what graduated, and what deliberately did not:**

Five durable decisions → [CONTEXT.md](../../CONTEXT.md), opening *Phase 4 — v1.3.8x
(2026)*. (The phase header is chronological, per the CONTEXT template — **not** named
after the docs program. The first draft called it "Phase 4 — Documentation
conformance", which would have implied the product entered a documentation era; it did
not, it is still shipping v1.3.8x.)

- **D90** two changelogs until the CI release gate is migrated
- **D91** two version spaces (extension vs web app) — declare the space, never mix the numbers
- **D92** the PRD needs a `⏸️ withdrawn` state — "shipped" is not forever
- **D93** `prd_ref` holds `FR-N` **or** `NFR-N` — requirement-centric, not FR-centric
- **D94** the one-time gapless ID renumber

Three traps → [LESSONS.md](../../LESSONS.md) §61–63, all three earned by costing this
backfill real time:

- **§61 merged ≠ enabled ≠ shipped** — containment is necessary, never sufficient; check the release *tree* and the build's env injection. (Bitcoin dApp injection commented out since v1.3.43; the NFT-mint service still instantiated in every release with an env var that is never set.)
- **§62 two release lineages, colliding version numbers** — extension 1.3.56 ≠ web-app 1.3.56.
- **§63 issue numbers collide with inherited upstream polkadot-js PRs** — date-window every `git log --grep`.

Source notes carry a `(graduated 2026-07-13)` marker. Three things stayed put on
purpose:

- `2026-07-07.md` §D (web-runner / webapp base coverage) and §E (mobile submodule) are
  **unexecuted proposals** — marking them graduated would imply a decision nobody made.
- `2026-07-09.md`'s later waves — same reason.
- `2026-07-13-id-renumber.md` **stays as a file**: D94 records the decision, but this
  note is the only place the full old→new ID mapping lives, and old commit messages
  and review comments still need it.
- `contributor-map.md` is not a scratch note at all — it is US-21.1's deliverable and
  the lookup table `assignee` resolution depends on. Labelled as such so a future
  close-out doesn't graduate it away.

**What this story does not close** (they are not conformance failures — they are the
owner's calls, recorded in [US-21.2](US-21.2-history-backfill.md)):

- the 5 PRD gaps (T&C display shipped with no FR; no requirement for data accuracy, UI
  responsiveness under scale, crash-resistance, or WebApp CSP);
- whether EPIC-21's docs stories can ever be `done` — the koni-docs done gate wants a
  `version_shipped` + CHANGELOG entry, and docs work ships in no product release. All
  three sit at `review`.
