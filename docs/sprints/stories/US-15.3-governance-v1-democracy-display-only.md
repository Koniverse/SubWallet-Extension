## Goal

On chains still running the legacy Governance V1 (Democracy) pallet rather than
OpenGov, a user can still *read* the active and past referenda — proposal,
status, and tally — inside the wallet, so the governance surface degrades
gracefully on legacy chains instead of showing nothing. This is a display-only
view: it intentionally exposes no vote action.

## Background

Before OpenGov, Polkadot/Kusama governance ran on the `democracy` pallet. Some
chains in the supported set have not migrated, so the OpenGov surface
([US-15.1](US-15.1-opengov-referenda-and-conviction-voting.md)) does not apply to
them. Rather than leave a blank governance tab on those chains, this story renders
the Democracy referenda **display-only**.

Democracy referenda data is aggregated through the same Services SDK backend
([AD-24](../../ARCHITECTURE.md#architecture-decisions)) as OpenGov, and this story
reuses the referenda subject fixture set up by US-15.1. It is **display-only by
design**: legacy Democracy voting is deliberately not wired up, since the chains
that matter are migrating to OpenGov and the product investment goes to the
OpenGov path. No vote / second / delegate extrinsic is exposed here, so there is
no write path and no key-signing surface in this story.

Materializes [FR-141](../../PRD.md#functional-requirements). This story is **retroactive** — already
shipped; `commit` / `version_shipped` are backfilled during version
reconciliation.

## Acceptance criteria

- [x] **AC-1** — **Given** an account on a chain still using the Governance V1
  (Democracy) pallet, **When** the user opens the governance surface, **Then** the
  Democracy referenda (proposal, status, tally) are listed and selectable for
  detail, sourced from the Services-SDK aggregation.
- [x] **AC-2** — **Given** the Democracy referenda view, **When** it is rendered,
  **Then** no vote / second / delegate action is exposed — the view is strictly
  display-only.
- [x] **AC-3** — **Given** a chain that exposes no Democracy referenda (or none
  active), **When** the user opens the view, **Then** an explicit empty state is
  shown rather than a blank or errored surface.

## Dev notes

### Architecture constraints

- [AD-24](../../ARCHITECTURE.md#architecture-decisions) — Democracy referenda data comes through the Services SDK backend, the same aggregation path as OpenGov; this view consumes it.
- This story does NOT introduce new AD entries and adds no write/signing path (display-only).

### Cross-story dependencies

- Builds on [US-15.1](US-15.1-opengov-referenda-and-conviction-voting.md) — reuses the referenda subject fixture and list/detail render pattern; this story specializes it to the Democracy shape.
- Sibling [US-15.4](US-15.4-opengov-delegation-and-governance-tracks.md) — both extend the governance surface; coordinate the shared referenda fixture.

### What we explicitly did NOT do

- No Democracy voting / seconding / delegation — display-only by design. Trigger to revisit: a still-active legacy chain requires write participation before its OpenGov migration.
