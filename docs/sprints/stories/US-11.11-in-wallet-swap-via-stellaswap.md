## Goal

A user can swap tokens on StellaSwap, the Moonbeam DEX, from inside the wallet,
with a live quote and single-confirm execution.

## Background

StellaSwap is the leading DEX on **Moonbeam** (an EVM-compatible Polkadot
parachain). It will be integrated as one **SwapService handler** (PRD
[FR-8](../../PRD.md#functional-requirements), EPIC-2) — a new provider is a new handler, never a new
swap-UI branch. Quote/fee data is aggregated via the Services SDK backend
([AD-24](../../ARCHITECTURE.md#architecture-decisions)); execution is an EVM
transaction on Moonbeam.

> **Note** — SubWallet's existing StellaSwap integration is liquid-staking
> ([EPIC-12](../epics/EPIC-12.md) / [US-12.4](US-12.4-liquid-staking-acala-bifrost-parallel-stellaswap.md));
> StellaSwap *swap* (this story) is forward-looking — no swap handler exists yet.

Materializes [FR-113](../../PRD.md#functional-requirements). 📋 **Planned** — forward-looking; not yet
started, tracked by [#2920 "Integrate Stellaswap SDK"](https://github.com/Koniverse/SubWallet-Extension/issues/2920)
(see also [#1793](https://github.com/Koniverse/SubWallet-Extension/issues/1793)).
The story is authored now so the provider slot is reserved in the locked FR
Coverage numbering.

## Acceptance criteria

- [ ] **AC-1** — **Given** a pair tradable on StellaSwap (Moonbeam), **When** the
  user enters an amount, **Then** a quote (rate, fee, slippage, min received) is
  shown.
- [ ] **AC-2** — **Given** a quote, **When** the user confirms, **Then** the swap
  transaction is signed, submitted, and tracked in history.
- [ ] **AC-3** — **Given** no route, slippage exceeded, or a stale quote, **When**
  the user swaps, **Then** a clear error is shown and nothing is submitted.

## Tasks

- [ ] **TASK-11.11.1** — StellaSwap handler producing Moonbeam quotes via SwapService (AC: 1)
- [ ] **TASK-11.11.2** — Build + sign + submit + track the EVM swap transaction (AC: 2)
- [ ] **TASK-11.11.3** — No-route / slippage-exceeded / stale-quote error states (AC: 3)
