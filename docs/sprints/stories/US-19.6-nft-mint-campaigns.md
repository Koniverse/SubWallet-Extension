## Goal

Users can discover and enter NFT-mint campaigns directly from the wallet: an active
mint campaign is displayed on the relevant in-app surfaces with its details and a
clear entry point to mint, so the team can run NFT drops to engaged users
release-free. When no campaign is active or the config is unavailable, nothing is
shown and the surface keeps working.

## Background

NFT-mint campaigns are remotely configured content delivered through the same
static-data / cache-proxy substrate as the other campaign surfaces
([AD-23](../../ARCHITECTURE.md#architecture-decisions),
[AD-25](../../ARCHITECTURE.md#architecture-decisions)). This story owns the
*display* of an NFT-mint campaign (artwork, details, mint CTA) and the *entry point*
to mint, reusing the remote-content mock and graceful-degradation pattern from
[US-19.4](US-19.4-in-app-campaign-banners.md). The mint transaction itself routes
through the existing transaction/NFT flows; this story surfaces the campaign and
hands off to mint.

Materializes [FR-157](../../PRD.md#functional-requirements). This story is **retroactive** — the capability
already ships in the product; `commit` / `version_shipped` are backfilled during
version reconciliation.

## Acceptance criteria

- [ ] **AC-1** — **Given** an active NFT-mint campaign in remote config, **When** the
  user opens the hosting surface, **Then** the campaign is displayed with its
  artwork/details and a mint entry point.
- [ ] **AC-2** — **Given** a displayed mint campaign, **When** the user taps mint,
  **Then** they are routed to the configured mint flow (the mint transaction is not
  re-implemented here).
- [ ] **AC-3** — **Given** no active campaign or a failed config fetch, **When** the
  surface renders, **Then** nothing is shown and the surface keeps working — the
  failure never breaks the host screen.
- [ ] **AC-4** — **Given** a campaign outside its active mint window, **When** the
  surface renders, **Then** the campaign is not offered as mintable.

## Implementation notes

**Stays `backlog` — and the reason I first wrote here was wrong.** (US-21.2, corrected 2026-07-13.)

The earlier note claimed "no candidate commit is contained in any extension tag".
**That is false.** The four Issue-1967 commits (Oct 2023, `7b92fd4955`, `c40e7faaae`,
`74f0bdba43`, `1b99af94ca`) *are* ancestors of every extension tag from **v1.1.36**
onward — `git merge-base --is-ancestor 7b92fd4955 v1.3.82` exits 0. A naive
`git tag --contains` re-trace would therefore "refute" this story and backfill
`version_shipped: 1.1.36`. Don't.

**What is actually true — containment is necessary but not sufficient.** The mint
*surface* was stripped **before** the first tag that contains those commits: the
Jan-2024 earning migration reset `extension-koni-ui` (`f7fc4b7` "[Issue-2361] reset
extension-koni-ui", `ec3fa2d`) and deleted the unlock-dot messaging, keeping it only
in `extension-web-ui`. And the extension build **never injects `NFT_MINTING_HOST`**
(only `.github/workflows/push-webapp-yield.yml` does), so `MINT_HOST` resolves to
`''` and the service cannot reach a mint host. At v1.3.82 every mint call in koni-ui
is commented out and `packages/webapp/public/images/subwallet/mint-nft-done.gif` is
an orphan referenced by nothing.

**Where it did ship:** the web-app **Earning Dashboard**, continuously deployed from
the `webapp-yield` branch, during the **Oct 24 – Nov 7 2023** campaign window — a CD
preview deploy, which produces **no version number in either space**. So
`version_shipped` stays empty: no numbered release, extension or web-app, ever
carried a working mint surface. Not `version_space: webapp` either — the web-app's
numbered releases don't have it.

Distinct from [US-19.7](US-19.7-mission-pools-reward-program.md): three separate
services exist at v1.3.82 — `mint-campaign-service` (this story, dead), plus
`campaign-service` and `mkt-campaign-service` (the general mission/banner surfaces,
which the extension *does* ship). No conflation.

**Cleanup candidate:** `MintCampaignService` is still instantiated in `State.ts` with
an env var that is never set, all UI callers commented out, and the mint asset
orphaned — dead code carried in every release since.
