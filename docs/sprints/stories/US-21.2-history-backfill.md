> **Note:** This is a 13-point story per the sizing scale — multi-week, cross-cutting,
> touching every story file, the CHANGELOG, the VERSION file, and the PRD. It is the
> largest single story in EPIC-21 and should be executed in order: CHANGELOG first,
> then VERSION, then story files, then PRD, then epics/sprints, and finally validate.

## Goal

Backfill the project's commit and version history into the doc layers so that
every `done` story has `version_shipped`, every CHANGELOG entry carries a real
commit SHA, version-progression entries exist for every intermediate version,
and VERSION is bumped to 1.3.82 to reflect the current state. Downstream epics
and the `koni-docs validate` gate depend on this history being complete.

## Background

The project has 169 stories across 20 epics, a CHANGELOG that goes to 1.3.79, and
a VERSION file at `1.3.79`. But:

- Many story files lack `version_shipped` in their frontmatter — it is either missing
  or set to `pending`.
- Many CHANGELOG entries use `pending` instead of a real commit SHA (RULE-2 violation).
- The VERSION file is at 1.3.79 while the product has shipped through subsequent
  releases (up to 1.3.82).
- Sprint archive files may be missing for earlier sprints.
- Epic statuses in PRD.md may be stale relative to actual story statuses.

This story closes all those gaps. It depends on US-21.1 for the `assignee` mapping
and feeds into US-21.3 for the final conformance check.

## Acceptance criteria

- [ ] **AC-1** — **Given** the CHANGELOG.md file, **When** `npx koni-docs backfill-commits` runs, **Then** every `pending` SHA is replaced with the real commit SHA from `git log`.
- [ ] **AC-2** — **Given** the VERSION file at 1.3.79, **When** current state is assessed, **Then** VERSION is bumped to 1.3.82 **And** the CHANGELOG has entries for 1.3.80, 1.3.81, 1.3.82 with real SHAs.
- [ ] **AC-3** — **Given** the 169 story files, **When** each `done` story is inspected, **Then** `version_shipped` is set to a bare semver (no `v` prefix) that matches the version in which it shipped.
- [ ] **AC-4** — **Given** the PRD.md FR tables, **When** every story's `prd_ref` is resolved, **Then** the FR status in PRD.md matches the story's `status` frontmatter.
- [ ] **AC-5** — **Given** the 20 epic files, **When** each epic's Stories table is checked, **Then** every story listed matches the actual story file's status and version_shipped.
- [ ] **AC-6** — **Given** the sprint archive, **When** every closed sprint is checked, **Then** every story referenced in the sprint scope table is accounted for.
- [ ] **AC-7** — **When** the backfill is complete, **Then** `npx koni-docs validate` exits zero.
- [ ] **AC-8** — **When** the user story's `assignee` is set, **Then** it matches the resolved GitHub login from US-21.1's contributor map (RULE-15).

## Dev notes — Architecture constraints

- RULE-16: `version_shipped` is bare semver — never `v`-prefixed. `0.3.1` ✅ — `v0.3.1` ❌.
- RULE-1: VERSION + CHANGELOG in same commit. Every time a version is bumped, the CHANGELOG entry goes in the same commit.
- RULE-2: CHANGELOG commit SHA mandatory, never `pending`.
- Use `git tag --list` to find version tags, then `git log --oneline tag1..tag2` to find commits per version.
