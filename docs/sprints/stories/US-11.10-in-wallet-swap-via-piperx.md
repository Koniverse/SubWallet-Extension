## Goal

A user can swap EVM tokens via PiperX from inside the wallet, with a live quote
and single-confirm execution.

## Background

PiperX is an EVM DEX provider, to be integrated as one **SwapService handler**
(PRD [FR-8](../../PRD.md#functional-requirements), EPIC-2) — a new provider is a new handler, never a new
swap-UI branch. Quote/fee data is aggregated via the Services SDK backend
([AD-24](../../ARCHITECTURE.md#architecture-decisions)); execution is an EVM
transaction.

Materializes [FR-112](../../PRD.md#functional-requirements). 📋 **Planned** — forward-looking; tracked by
[#4024 "Support PiperX swap"](https://github.com/Koniverse/SubWallet-Extension/issues/4024)
(open). No PiperX handler exists in code yet.

## Acceptance criteria

- [ ] **AC-1** — **Given** an EVM pair tradable on PiperX, **When** the user enters
  an amount, **Then** a quote (rate, fee, slippage, min received) is shown.
- [ ] **AC-2** — **Given** a quote, **When** the user confirms, **Then** the swap
  transaction is signed, submitted, and tracked in history.
- [ ] **AC-3** — **Given** no route, slippage exceeded, or a stale quote, **When**
  the user swaps, **Then** a clear error is shown and nothing is submitted.

## Tasks

- [ ] **TASK-11.10.1** — PiperX handler producing quotes via SwapService (AC: 1)
- [ ] **TASK-11.10.2** — Build + sign + submit + track the EVM swap transaction (AC: 2)
- [ ] **TASK-11.10.3** — No-route / slippage-exceeded / stale-quote error states (AC: 3)
