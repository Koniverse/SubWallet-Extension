## Goal

A user can swap EVM tokens via KyberSwap — an EVM DEX aggregator that sources the
best route across many underlying pools — from inside the wallet, with a live
quote and single-confirm execution.

## Background

KyberSwap is an **EVM aggregator**: rather than a single pool, it routes across
many DEXes to find the best execution. It is integrated as one **SwapService
handler** (PRD [FR-8](../../PRD.md#functional-requirements), EPIC-2) — a new provider is a new handler,
never a new swap-UI branch — and quote/fee data is aggregated via the Services
SDK backend ([AD-24](../../ARCHITECTURE.md#architecture-decisions)). KyberSwap is
a **keyed provider**: its aggregator API is proxied through the SubWallet backend
per [NFR-16](../../PRD.md#non-functional-requirements), never shipped in the bundle. Execution is an EVM
transaction.

Materializes [FR-106](../../PRD.md#functional-requirements). **Retroactive** — already shipped.

## Acceptance criteria

- [ ] **AC-1** — **Given** an EVM pair routable on KyberSwap, **When** the user
  enters an amount, **Then** an aggregated quote (best route, rate, fee, slippage,
  min received) is shown.
- [ ] **AC-2** — **Given** a quote, **When** the user confirms, **Then** the swap
  transaction is signed, submitted, and tracked in history.
- [ ] **AC-3** — **Given** no route, slippage exceeded, or a stale quote, **When**
  the user swaps, **Then** a clear error is shown and nothing is submitted.
