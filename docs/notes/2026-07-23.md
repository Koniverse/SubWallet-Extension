### The shape of the area

**78 of 81 issues are Ledger; 3 are Keystone; Polkadot Vault has none.** The QR transport has no
cable to disconnect, no firmware to fall behind, and no per-chain app to keep current. The USB one
has all three — and one Ledger-specific programme, the Generic app, accounts for 14 rows on its own.

**Vara is the whole argument for the Generic app in two rows.** #2283 *"Support Ledger for the Vara
network"* closed `NOT_PLANNED`; two years later #3835 shipped Vara in 1.3.9 **through the Generic
app**. The per-chain route was abandoned; the platform route delivered.

**Five rounds of "update the runtime metadata for the Ledger app"** (#4480, #4653, #4715, #4828,
#4876) and **not one carries a CHANGELOG line.** A chain upgrades its runtime, the Generic app's
metadata goes stale, the device can no longer render what it is asked to sign. The work ships
inside releases without being named, which is why the ledger showed every round as `—`.

### A CHANGELOG typo created a phantom row and hid a real release

The ledger's US-36.61 read:

> `#3902 — "Optimize swap pair selection" — 📋 backlog`

**#3902 is not that issue.** Its tracker title is *"[WebApp] Check the `@polkadot/api` library
version for signing with Ledger"* — a real, open, Ledger issue. The title came from
[docs/CHANGELOG.md:881](../CHANGELOG.md), which delivers *"Optimize swap pair selection"* in 1.3.30
and **mis-numbers it `(#3902)` when the issue is [#2902](https://github.com/Koniverse/SubWallet-Extension/issues/2902)**.

The generator trusted the bullet. Two consequences, one in each direction:

1. A Ledger issue was filed in the hardware ledger under a swap issue's name — **fixed here**, the
   row is in [US-16.10](../sprints/stories/US-16.10-ledger-on-webapp-and-avail-space.md) with the
   tracker's title.
2. `US-31.18`, which correctly owns **#2902**, shows `Shipped: —` — because the version matcher
   looked for `(#2902)` and the line says `(#3902)`. **The release exists: 1.3.30.** Not fixed here;
   US-31.18 belongs to the swap ledger, which is still unfolded. **Flag for the EPIC-31 fold.**

A one-digit typo in a changelog bullet is enough to both invent a row and erase a version. Neither
failure is visible from inside the generated docs — only the tracker disagrees.
