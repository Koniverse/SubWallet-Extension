# Discussion notes — 2026-06-11

Outcome of a PRD-restructuring discussion. These are **proposals for the next
round** — not yet applied to `docs/PRD.md`.

## A. Epic / group restructuring (proposed)

1. **Reorder** — move `chain-management` above `security` in the epic ordering.
2. **Merge** — fold `chain-abstraction` into `chain-management` (one epic).
3. **Split groups** — break the combined **DeFi & Governance** group into two
   separate groups (DeFi; Governance).
4. **NFR → epic (when generating epics)** — during epic generation, read the
   NFR section and derive a fitting epic from it (e.g. a performance-optimization
   / hardening epic) so the NFRs get real epic + story coverage, instead of
   sitting as a flat NFR list with no owner.
5. **New "Utilities" group** — regroup `onboarding`, `campaign`, and
   `notification` under a single Utilities group.

## B. Decomposition guidance

6. When breaking down: aim for **3 epics spanning 3 different groups**, with
   **2 user stories per epic**.

## C. Authoring principle

7. For existing/old content, **generalize and abstract** it so the AI keeps
   enough context to handle the subsequent tasks (don't drop information when
   condensing).

## D. Constraints / open concerns

8. **Three-platform burden** — supporting extension + web app + mobile at once
   is hard; keep in mind when scoping the restructure.
